- `pages/index/index.js`：onLoad 中从 `app.globalData.subscriptions` 取数据设到 page data
- `pages/index/index.wxml`：改用 `wx:for="{{subscriptions}}"` 动态渲染，不再手写 sources 值

## 2026-05-19 修复 CF Workers & GitHub Pages 界面问题

### 问题1：CF Workers 无图标版没有链接
- **根因**：`src/worker.js` `renderHTML()` 中 `iconMode` 初始值为 `'no-icon'`，
  但 JS `dataset['no-icon']` 无法访问 `data-noicon` 属性（正确访问是 `dataset.noicon`）。
- **修复**：将 `iconMode` 改为 `'noicon'`（与 `dataset.noicon` 对应），
  `toggleIconMode()` 中同步修改。

### 问题2：GitHub Pages 界面没有变化
- **根因**：`.github/workflows/generate.yml` 工作流每次运行时会调用
  `node src/generate-index.js` 重新生成 `output/index.html`，
  该脚本内部的旧模板把手动修改的版本覆盖回旧版。
- **修复**：重写 `src/generate-index.js`，使其生成与新版一致的正确 HTML
  （图标切换开关、正确分区结构、单链接 `data-icon/data-noicon` 模式）。
- **关键**：`src/generate-index.js` 是 GitHub Pages 界面的"真实来源"，
  修改界面时必须同步更新此脚本，否则会被自动提交覆盖。

### 提交记录
- `da57713` fix(worker): 修复 renderHTML 中 iconMode 变量名
- `3c31f61` fix: 重写 generate-index.js 使其生成与新版一致的 index.html
