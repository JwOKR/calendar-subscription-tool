### 3. 修复 Cloudflare Workers 日历合并 bug（本轮会话）

#### Bug 1: 事件重复
- 原因：`addEvent()` 无去重逻辑，不同源（holidays/festivals/lunar）产生同名同日事件会叠加
- 修复：新增 `createEventCollector()` + `dedupeKey()`，基于 `date|summary|endDate` 三元组去重
- 影响：`src/worker.js`

#### Bug 2: 不连续假期被合并成超大事件
- 原因：`parseTimorFormat` 合并条件只检查 `holiday===true`，不检查日期连续性
- 导致清明节（4/4~4/6）和劳动节（5/1~5/5）被合并为一个横跨 4/4~5/5 的事件
- 修复：添加 `nextDate.diff(currentEndDate.add(1,'day'), 'day') === 0` 日期连续性检查
- 影响：`src/worker.js` + `api/calendar.js`（CLI 版 `fetch-holidays.js` 原本有此检查，无此 bug）

#### Bug 3: 所有年份返回相同节假日数据
- 原因：默认 API URL `https://timor.tech/api/holiday/year` 没有 `{year}` 占位符
- `.replace('{year}', year)` 无效，所有年份请求同一 URL，永远返回当年数据
- 修复：默认 URL 改为 `https://timor.tech/api/holiday/year/{year}`
- 影响：`src/worker.js` + `api/calendar.js` + `src/fetch-holidays.js`（CLI 版也有此 bug）
